## 议题一：语音功能闪退（问题1&3）— 🔴 紧急 P0

### 问题描述
语音唤醒/命令词识别功能在部分场景下闪退，疑似权限声明缺失和线程安全问题。

### 决议方案
1. **权限声明**：检查 `AndroidManifest.xml` 是否已声明 `RECORD_AUDIO` 权限，Flutter 端运行时通过 `permission_handler` 库动态申请
2. **线程安全**：`VoicePlugin.kt` 中的 `initEsrEngine` 从主线程移到子线程（使用 Kotlin Coroutine）
3. **异常兜底**：Native 端所有关键节点加 `try-catch`，捕获异常后通过 `MethodChannel` 返回 `false` 而非直接抛异常
4. **前端保护**：Dart 端 `VoiceService.init()` 增加 fail-safe 机制，初始化失败时 UI 层显示"语音功能不可用"提示而非白屏

**负责人：** 张工 + 老陈
**预估工时：** 2 天
**截止时间：** 本周五前

## 议题二：拍照识别药品信息（问题2）— 🟡 重要 P1

### 问题描述
用户需要拍照识别药品名称和规格，自动填入添加药品表单，减少手动输入。

### 决议方案
1. **OCR 引擎选型**：采用 **华为 ML Kit（Text Recognition SDK）**
   - 离线运行，不依赖网络
   - 华为手机上识别速度极快，功耗低
   - 上架华为应用市场是必经之路，用华为 SDK 有生态优势
   - 集成方式：Gradle 依赖 + Flutter MethodChannel 桥接
   - 备选方案：百度 OCR 离线版（跨平台一致性好，但非华为系）
   - 已排除 Google ML Kit（国内手机 Google 服务不一定可用）
2. **交互流程**：拍照 → OCR 识别 → 用户确认 → 自动填入表单（不做全自动，第一期先用户确认再填入）

**负责人：** 张工
**预估工时：** 5-7 天

## 议题三：用药频率与提醒时间（问题4）— 🟡 重要 P1

### 问题描述
当前添加药品时无法灵活设定一天多次的用药频率及对应的提醒时间。

### 决议方案
1. **UI 交互**：
   - "一天几次"：滑动选择器或大按钮组（1次/2次/3次/4次）
   - 选择后下方动态展开对应数量的时间选择器（时：分）
2. **后端**：调用现有 `add` 接口后再调批量 `create schedule` 接口；确认 `medication_schedules` API 是否支持批量写入，不支持则补充
3. **数据模型**：`MedicationSchedule` 类扩充以支持多时间点

**负责人：** 张工（前端）+ 老陈（后端）
**预估工时：** 3 天

## 议题四：离线可用方案（问题5）— 🟡 重要 P1

### 问题描述
用户在无网络环境下无法查看药品信息、打卡记录等，需要离线缓存能力。

### 决议方案
1. **本地数据库选型**：采用 **sqflite（SQLite）**
   - 关系型数据库，药品/日程/打卡数据天然适合表结构
   - Flutter 生态成熟，社区文档完善
2. **缓存优先级**：① 药品数据离线缓存（最高优先级）② 打卡记录 ③ 积分
3. **同步策略**：
   - 读取：先查本地缓存 → 有网时后台刷新
   - 写入：同时写入本地 + 远端，保证数据一致
4. **新增组件**：
   - `LocalDatabaseService` — 本地数据库层
   - `ApiService` 增加缓存读写逻辑

**负责人：** 张工（本地数据库层）+ 老陈（同步逻辑）
**预估工时：** 5-7 天（一期仅药品数据离线）
