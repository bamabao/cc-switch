## 二、页面详细设计

---

### P01 — 启动页（Splash）

| 项目 | 内容 |
|------|------|
| **用途** | APP 冷启动品牌展示 + 初始化检查 |
| **核心元素** | ① APP Logo（爸妈宝大字） ② 副标题"您的用药管家" ③ 版本号 ④ 加载进度条 |
| **适老设计** | ① Logo 字体 ≥ 48sp，副标题 ≥ 28sp ② 纯色高对比背景（深蓝底白字） ③ 停留时间 ≥ 2.5s（老人阅读慢） ④ 加载超时自动进入语音引导页 |
| **核心交互** | 无操作自动跳转；加载失败显示"网络不太顺畅"语音提示 + 重试按钮 ≥ 80×80px |
| **语音兜底** | 无（启动时语音引擎尚未就绪） |

---

### P02 — 语音引导页（首次启动 / 语音唤醒）

| 项目 | 内容 |
|------|------|
| **用途** | 首次启动引导 + 语音唤醒入口。非首次可跳过 |
| **核心元素** | ① 大号麦克风动画图标（≥ 120×120px） ② "对我说"提示文字 ③ 示例语音命令（"帮我看今天的药""我的积分有多少"） ④ "跳过"按钮 ⑤ "不再显示"勾选框 |
| **适老设计** | ① 全屏单页设计，零干扰 ② 提示文字 ≥ 36sp ③ 示例命令按钮可点击，点一下自动说出该命令 ④ 按钮 ≥ 90×64px |
| **核心交互** | 点击 \> 进入语音识别状态；点击示例命令 \> 直接执行；点击跳过 \> 进入首页；勾选"不再显示" \> 下次启动跳过 |
| **语音兜底** | 本页即为语音兜底入口，全程语音驱动 |

---

### P03 — 首页（Dashboard）

| 项目 | 内容 |
|------|------|
| **用途** | 老人打开 APP 看到的第一个页面，今日核心信息聚合 |
| **核心元素** | **① 顶部问候区**：当前时间 + 天气 + 老人称呼（"王大爷，早上好！"） **② 用药概览卡片**（最重要）：今日待服药数量 + 已服/未服状态 + 下次服药倒计时 **③ 快捷操作区**：3 个大按钮横排 — 【扫一扫】【语音助手】【紧急求助】 **④ 积分卡片**：当前积分 + 今日签到赚积分入口 **⑤ 今日提醒**：药品到期、复诊提醒等 |
| **适老设计** | ① 问候语 ≥ 32sp，药品名称 ≥ 36sp，时间数字 ≥ 48sp ② 高对比卡片（白底黑字 / 黄底黑字） ③ 每个卡片区域 ≥ 120px 高 ④ 卡片之间用粗色块分隔（8px 灰色/浅蓝） ⑤ 待服药区用红色/橙色高亮警示 ⑥ 页面可上下滑动，但核心信息折叠不超过 3 屏 |
| **核心交互** | 点击用药卡片 → P07 药品详情；点击"扫一扫"按钮 → 打开相机扫码页；点击"语音助手" → 唤醒语音；点击"紧急求助" → SOS 页；点击积分卡片 → 积分明细页 |
| **语音兜底** | 完全语音命令支持：说出"今天吃什么药""我的积分""帮我扫一扫"等 → 自动跳转并执行 |

---

### P04 — 药品列表

| 项目 | 内容 |
|------|------|
| **用途** | 显示老人所有已添加药品 |
| **核心元素** | ① 顶部搜索框（语音输入+文字输入） ② 药品列表（卡片布局） 每张卡片包含：药品名（大字）、规格、剩余数量、下次服药时间、颜色标记（红=即将到期/已过期、绿=正常、灰=已停药） ③ 底部大号【+ 添加药品】浮动按钮（≥ 80×80px） |
| **适老设计** | ① 药品名称 ≥ 36sp，辅助信息 ≥ 24sp ② 卡片最小高度 140px，整张卡片可点击 ③ 远处可辨状态：用整体卡片底色标示（过期=淡红底、正常=白底） ④ 每张卡片右侧配药品照片（若有） ⑤ 进入页面自动语音播报："您有X种药品，其中Y种今天需要服用" |
| **核心交互** | 点击卡片 → 药品详情；长按卡片 → 弹出操作菜单（编辑/删除/设为停药）；点击"+" → 添加药品页；点击搜索 → 输入或语音搜索 |
| **语音兜底** | 说出药品名 → 自动跳转到该药品详情；说出"添加新药" → 跳转扫码页 |

---

### P05 — 添加药品（扫码）

| 项目 | 内容 |
|------|------|
| **用途** | 通过扫描药盒条码自动识别药品信息 |
| **核心元素** | ① 全屏相机预览（扫码框在中央） ② 大号提示文字："将药盒条形码对准框内"（≥ 28sp，白字黑底） ③ 底部操作栏：【手动输入】【手电筒】【返回】三个大按钮 |
| **适老设计** | ① 扫码框 ≥ 屏幕宽 70%，便于老人对位 ② 按钮 ≥ 80×80px，图标+文字双标识 ③ 成功识别时发出"嘀"声 + 振动 + 语音播报药名 ④ 扫码失败提示语友善："没扫到，可以试试点'手动输入'" |
| **核心交互** | 扫码成功 → 自动查询药品数据库 → 进入药品信息确认页 / 直接跳转药品详情（自动填充）；扫码失败 → 提示重试或转手动输入 |
| **语音兜底** | 说出"手动输入"→ 跳转手动录入页；说出"打开手电筒"→ 开启闪光灯 |

---

### P06 — 添加药品（手动录入）

| 项目 | 内容 |
|------|------|
| **用途** | 手动录入药品信息（扫码失败或新增非标准药品） |
| **核心元素** | 表单字段（每字段一个完整行，大号输入）： ① 药品名称（语音输入+键盘） ② 药品照片（相机拍照/相册选取） ③ 规格（如"0.25g×48片"） ④ 用法用量（语音输入+选择器） ⑤ 服药频次（视觉化选择：每天X次，每次X粒/片） ⑥ 提醒时间（大号时间选择器，支持多个时间点） ⑦ 疗程/有效期 ⑧ 备注（语音输入） ⑨ 底部【保存】【取消】两大按钮 |
| **适老设计** | ① 每字段标签 ≥ 28sp，输入内容 ≥ 32sp ② 输入框最小高度 80px ③ 时间选择器用大号滚轮（格≥60px高） ④ 频次选择用可视化图示（太阳=早上、月亮=晚上，搭配大图标） ⑤ 每填写完一个字段自动语音提示下一字段 ⑥ 支持全程语音填写（"名字叫阿莫西林，每天三次，每次一粒"） |
| **核心交互** | 点击输入框 → 弹出大号键盘或语音输入；点击拍照 → 打开相机；保存 → 跳转药品详情页（P07）；取消 → 返回药品列表 |
| **语音兜底** | 本页面全字段支持语音输入；说出"保存"→ 保存并提交；"不填了"→ 取消返回 |

---

### P07 — 药品详情

| 项目 | 内容 |
|------|------|
| **用途** | 查看单个药品的完整信息，今日服药状态和操作 |
| **核心元素** | **① 顶栏**：药品名（超大，≥ 40sp）+ 药品照片 **② 今日服药状态**（最重要区域）：大号四象限或时间轴，每个时间点标明★已服/○未服，当前时间点高亮闪烁催促 **③ 服药操作区**：【我已服药】大按钮（≥ 100×60px，绿色），点击记录当前时间 **④ 基本信息区块**：规格、用法用量、剩余数量、有效期、备注 **⑤ 底部操作栏**：【编辑】【更改提醒】【删除药品】【扫药盒确认】四个大按钮 |
| **适老设计** | ① 核心操作（"我已服药"按钮）置于屏幕中上黄金位置，颜色突出（绿色高亮） ② 时间点用图形化表盘/时间轴呈现，而非文字列表 ③ 剩余数量显示用进度条（横条从满到空） ④ 过期药品整体卡片变红色并显示 ⚠ 过期警告 ⑤ 进入页面自动语音播报："现在是X点，您需要在Y点服用Z药，每次X粒" |
| **核心交互** | 点击"我已服药"→ 记录服药时间 + 振动反馈 + 语音确认"已记录"；点击时间点 → 显示该次记录详情；点击编辑 → 跳转修改页；点击扫药盒确认 → 打开扫码确认是否是此药 |
| **语音兜底** | 说出"我吃过药了/已服药"→ 记录；说出"这个药怎么吃"→ 语音播报用法用量；"帮我拍个照"→ 打开相机更新药品照片 |

### P08 — 服药记录（今日/详情）

| 项目 | 内容 |
|------|------|
| **用途** | 查看某一药品今日/历史每次服药记录的详细日志 |
| **核心元素** | ① 大号日历时间轴：显示今天/昨天的服药时间点 ② 每次记录卡片：服药时间、是否按时、备注（如有不适） ③ 【补录】按钮：老人如果忘记点击"已服药"，可以补录 ④ 【漏服建议】区域：如果某次漏服，显示补服建议（"您可以在X点前补服"） |
| **适老设计** | ① 时间轴用大图标：✓按时 / ⚠ 延迟 / ✗ 漏服 ② 每条记录卡片 ≥ 100px 高 ③ 补录按钮闪烁提醒（如果有漏服） ④ 语音播报："今天X点吃了Y药，记录正常" |
| **核心交互** | 左右滑动切换日期；点击记录 → 展开详情（含备注）；点击补录 → 选择时间点并记录；点击"漏服建议"→ 语音+文字展示补服方案 |
| **语音兜底** | "我今天吃药了吗"→ 语音播报今日服药情况；"我忘吃药了怎么办"→ 给出补服建议 |

### P09 — 服药日历/历史记录

| 项目 | 内容 |
|------|------|
| **用途** | 全局视角查看多天多药品的服药完成情况 |
| **核心元素** | ① 大号日历视图（只显示当月，每格 ≥ 80×80px） ② 每日完成率（格子颜色：全绿=100%、黄=部分完成、红=漏服较多、灰=无记录） ③ 点击某天 → 展开当日详细服药时间线 ④ 月度统计：本月按时率（%） |
| **适老设计** | ① 日历格子巨大，手指可直接点击哪一天 ② 每天标注简化：一个数字（完成百分比） + 颜色面 ③ 月度统计用大号数字+进度条 ④ 进入页面语音概览："这个月您一共按时服药XX次，按时率是XX%" |
| **核心交互** | 点击日期 → 展开详情列表；左右滑动 → 切换月份；长按某天 → 语音播报当天情况 |
| **语音兜底** | "上个月我吃药怎么样"→ 显示并语音播报上月统计；"这周哪几天没吃"→ 过滤显示漏服日 |

### P10 — 用药提醒设置

| 项目 | 内容 |
|------|------|
| **用途** | 设置/修改某一药品的提醒方式和时间 |
| **核心元素** | ① 当前提醒时间列表（大号时间块，可拖动排序） ② 【添加提醒时间】按钮 ③ 提醒方式选择：铃声 / 振动 / 语音播报（可多选） ④ 提前提醒时间：服药前 5/10/15/30 分钟 ⑤ 重复提醒间隔：每 5/10/15 分钟一次，共提醒 3 次 ⑥ 子女同步提醒开关（开启后提醒信息同步推送到子女端 APP） |
| **适老设计** | ① 时间选择器用特大滚轮（格高 60px） ② 开关按钮 ≥ 60×40px，配阴阳色（绿=开/灰=关） ③ 每次操作都有语音确认 ④ 提醒铃声可选老人喜欢的（戏曲、民歌、经典老歌片段） |
| **核心交互** | 点击加号 → 弹窗添加新提醒时间；拖动调整顺序；开关控制各功能；保存 → 测试播放一次提醒效果 |
| **语音兜底** | "帮我加一个晚上8点的提醒""把提醒声音换成京剧" |

### P12 — 商城首页（积分兑换）

| 项目 | 内容 |
|------|------|
| **用途** | 用积分兑换商品的积分商城 |
| **核心元素** | **① 积分余额横幅**（最上方）：当前积分 + 今日签到可领积分 + 进度条显示"还差XX分可兑换热门商品" **② 商品分类**：大号横排标签（全部 / 家用 / 健康 / 食品 / 药品优惠券） **③ 商品瀑布流**：每格大号卡片（≥ 240×300px），含商品图、名称、所需积分、剩余数量 **④ 底部导航固定栏** |
| **适老设计** | ① 积分数字 ≥ 48sp（超大），让老人一眼看到自己有多少积分 ② 商品卡片：商品图占 60%、文字 40%，文字 ≥ 28sp ③ 每格卡片都有一个"积分"标签，不显示人民币 ④ 低库存商品红色角标"仅剩X份" ⑤ 进入页面自动语音播报积分余额 |
| **核心交互** | 点击商品卡片 → 商品详情；点击签到 → 领取每日积分+动画反馈；分类标签左右滑动切换；下拉刷新 |
| **语音兜底** | "我有多少积分""这个月签到领了多少分""看看有什么可以换的" |

### P13 — 商品详情

| 项目 | 内容 |

## 五、适老设计要点汇总

### 5.1 视觉
| 设计项 | 标准 |
|--------|------|
| 正文字体 | ≥ 28sp（普通 APP 一般 14-16sp） |
| 标题字体 | ≥ 36sp |
| 关键数字 | ≥ 48sp（积分、时间、日期） |
| 按钮最小尺寸 | 80×60px（核心按钮 100×56px） |
| 点击热区 | ≥ 80×80px |
| 对比度 | 最低 7:1（黑字黄底 或 白底黑字） |
| 配色 | 禁用低饱和、邻近色搭配；高饱和纯色为主 |
| 图标 | 功能 icon 必须配文字说明 |
| 行间距 | ≥ 1.6 倍行高 |

### 5.2 交互
| 设计项 | 标准 |
|--------|------|
| 滑动 | 支持慢速滑动，惯性减少 |
| 点击 | 所有元素可点击区域 ≥ 可视化区域 |
| 手势 | 禁用复杂手势（双指缩放等限制使用）；主要交互只有"点击" |
| 反馈 | 每次操作必须有反馈：振动 + 声音 + 视觉三者至少两种 |
| 误操作 | 关键操作必须二次确认（语音 + 点击） |
| 超时 | 页面加载/操作超时有明确提示和兜底 |
| 取消 | 所有操作都可取消（回到上一状态） |

### 5.3 语音
| 设计项 | 标准 |
|--------|------|
| 覆盖范围 | 所有页面、所有核心操作都支持语音命令 |
| 唤醒 | 始终监听唤醒词（低功耗模式） |
| 语音播报 | 进入核心页面自动播报关键信息 |
| 方言 | 优先支持当地方言（需接入支持方言的语音引擎） |
| 语速 | 可调节（慢/中/正常），默认"慢" |
| 回退 | 语音识别失败时提示老人重试或转文字输入 |
| 多轮对话 | 支持上下文连续对话 |

### 5.4 无障碍
| 设计项 | 标准 |
|--------|------|
| 系统辅助功能 | 兼容 Android TalkBack / iOS VoiceOver |
| 高对比度模式 | 提供独立的高对比主题切换 |
| 触摸放大 | 可开启触摸位置放大镜 |
| 色觉障碍 | 信息传达不依赖颜色（颜色仅作为辅助） |
