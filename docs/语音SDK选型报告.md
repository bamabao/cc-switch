# 爸妈宝 · 语音 SDK 选型对比报告

> 生成日期：2026-06-29  
> 目标平台：Android + iOS（Flutter 跨平台）  
> 核心需求：老人端语音录入药品/剂量、方言兼容、普通话 TTS 播报、离线兜底

---

## 一、需求要点回顾

| # | 需求 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 语音录入药品名称、剂量（ASR） | P0 | 老人说"阿莫西林胶囊，一天两次，一次一粒"→ 解析为结构化数据 |
| 2 | 方言兼容 | P0 | 粤语、四川话、东北话等；老人普通话不标准也需识别 |
| 3 | 普通话 TTS 播报 | P0 | 清晰读出"药品名+用量"确认 |
| 4 | 离线兜底 | P1 | 无网络时能用简单命令（"确认""取消""大声点"） |
| 5 | Flutter 集成 | P0 | 需有原生插件或 Dart SDK，Android/iOS 双端覆盖 |

## 三、详细分析

### 3.1 科大讯飞 — 推荐首选 🏆

#### 优势
- **方言最强**：30+ 方言，且支持"带方言口音的普通话"混合场景——这对老人非常关键（川普、广普都能识别）
- **离线 SDK 成熟**：可直接在 Android/iOS 端加载离线语音资源包，无网时识别药品名+简单命令
- **TTS 体验好**：老人听力弱，可选择语速慢+大音量发音人，体验远超其他方案
- **行业口碑**：医院、药房场景有大量落地案例

#### 劣势
- 离线 SDK 需付费授权（按年），但报价可谈
- Flutter 没有官方插件，需自行封装 platform channel
- 注册/审核流程略复杂

#### 定价参考（2025-2026）
| 项目 | 价格 |
|------|------|
| 实时语音识别 | 3.7 元/万秒 |
| 离线命令词 SDK | 约 1 万/年（基础版） |
| 离线语音听写 | 约 3 万/年 |
| TTS 合成 | 3.0 元/万次 |

> **Tips**：如果只用在线 + 少量离线命令词，完全够用免费额度；若需大规模离线听写再买授权。

---

### 3.2 阿里云语音识别 — 备选方案 🥈

#### 优势
- **Flutter 支持最好**：官方 Flutter SDK（`aliyun_nls_sdk`），接入文档清晰
- **TTS 效果优秀**：阿里云语音合成的自然度已接近讯飞
- **价格便宜**：实时 ASR 2 元/万秒，比讯飞便宜近一半
- **阿里生态**：如果用阿里云服务器，内网调用延迟更低

#### 劣势
- **无离线方案**：必须联网；如果要离线，需搭配 sherpa-onnx 等第三方引擎，增加集成复杂度
- **方言覆盖不如讯飞**：仅 10+ 种，且部分方言识别准确率有差距
- **离线+在线切换逻辑需自写**：网络切换的兜底体验不如讯飞原生流畅

#### 定价参考
| 项目 | 价格 |
|------|------|
| 实时语音识别（按量） | 2.0 元/万秒 |
| 录音文件转写 | 1.5 元/小时 |
| 语音合成（标准） | 2.0 元/万次 |
| 语音合成（精品） | 3.0 元/万次 |

---

### 3.3 微信原生语音 — 不适合 ❌

#### 分析
- **强绑定微信环境**：`wx.getVoicerecognizer` 仅在微信小程序/公众号 JS-SDK 可用
- **Flutter APP 无法直接使用**：除非在 Flutter 中内嵌 WebView 跑微信小程序——体验差、不可控
- **方言有限**：仅粤语+四川+东北常见方言，其他方言覆盖差
- **TTS 较弱**：微信同声传译插件音色少，不适合药名播报这种需要清晰度的场景

**结论**：本项目是独立 APP（非微信小程序），微信语音方案**完全不适用**。

### 核心流程


## 五、推荐结论

### ✅ 推荐方案：**科大讯飞**（在线 ASR + TTS + 离线命令词 SDK）

**理由：**

1. **方言覆盖最广**（30+ 种，含粤语、四川话、东北话），且有「带方言口音的普通话」混合识别——爸妈宝目标用户是老人，这一项是刚需
2. ****唯一有成熟离线 SDK** 的厂商**——离线命令词、离线听写都可用，不必额外集成第三方离线引擎
3. **TTS 质量高**——发音人多、可调语速/音量，适合老年人聆听体验
4. **行业验证**——医疗/药房场景大量使用，稳定性有保障
5. **Flutter 集成可行**——虽无官方插件，但通过 method channel 封装是成熟做法，如有需要可自行封装或使用社区插件

### 备选方案：**阿里云语音识别 + sherpa-onnx 离线**

- 如果团队 Flutter 经验强、看重官方 Flutter SDK 支持
- 离线需求小（仅简单命令）时可以使用 sherpa-onnx 作为互补
- 成本更低

### ❌ 不推荐：**微信原生语音**

- 与 Flutter APP 架构不兼容
- 方言覆盖和 TTS 质量都有限
- 除非项目改为微信小程序，否则不采用

## 六、接入步骤（讯飞方案）

### Step 1：注册 & 申请应用

1. 前往 [讯飞开放平台](https://www.xfyun.cn/)
2. 注册开发者 → 创建应用 → 选择服务：
   - **语音听写**（ASR，用于语音输入）
   - **语音合成**（TTS，用于播报）
   - **离线命令词识别**（可选）
3. 获取 `APPID`、`APIKey`、`APISecret`

### Step 2：下载 SDK

- Android：[讯飞 MSC SDK (Android)](https://www.xfyun.cn/sdk/dispatcher)
- iOS：[讯飞 MSC SDK (iOS)](https://www.xfyun.cn/sdk/dispatcher)
- 选择对应平台的 SDK 包，解压到 Flutter 项目的 `android/` 和 `ios/` 目录

### Step 3：Flutter 端封装 Method Channel

```dart
// flutter/lib/services/voice_service.dart

import 'package:flutter/services.dart';

class VoiceService {
  static const _channel = MethodChannel('com.bamaobao/voice');

  /// 开始语音识别（在线，自动识别方言）
  static Future<String> startListening() async {
    return await _channel.invokeMethod('startListening');
  }

  /// 离线命令词识别（无网络时）
  static Future<String> startOfflineListening() async {
    return await _channel.invokeMethod('startOfflineListening');
  }

  /// TTS 播报文本
  static Future<void> speak(String text) async {
    return await _channel.invokeMethod('speak', {'text': text});
  }

  /// 停止播报
  static Future<void> stopSpeaking() async {
    return await _channel.invokeMethod('stopSpeaking');
  }
}
```

### Step 4：Android 原生实现

```kotlin
// android/app/src/main/kotlin/.../MainActivity.kt

class MainActivity : FlutterActivity() {
    override fun configureFlutterEngine(flutterEngine: FlutterEngine) {
        super.configureFlutterEngine(flutterEngine)
        VoicePlugin().registerWith(flutterEngine.dartExecutor.binaryMessenger)
    }
}
```

## 七、关键风险 & 应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 网络不稳定，ASR 无响应 | 老人无法录入 | 离线命令词兜底 + 缓存最近 5 条药品名到本地 |
| 方言识别不准 | 药品名录错 | 加入"双确认"流程：TTS 回读药品名 → 老人语音确认/纠正 |
| 老人说话语速慢/口齿不清 | 超时断句 | 调长讯飞 VAD 超时时间（建议 2000ms+），且允许手动按结束按钮 |
| 离线 SDK 授权到期 | 离线功能失效 | 提前 30 天采购续约；设本地 fallback（文本列表选择） |
| Flutter 原生桥接 bug | 集成进度延迟 | 预留 1-2 周调试窗口；先做 Android 版验证再拓展到 iOS |

## 八、备用参考：sherpa-onnx 离线方案

> 如果预算有限不想买讯飞离线授权，可考虑 sherpa-onnx 作为离线补充

| 项目 | 说明 |
|------|------|
| 项目地址 | https://github.com/k2-fsa/sherpa-onnx |
| 支持引擎 | 支持 Chinese ASR（基于 Paraformer / Zipformer / Whisper 模型） |
| Flutter 集成 | 有 `sherpa_onnx` Flutter 插件（社区，非官方） |
| 方言 | 需加载对应方言模型（体积较大，约 30-50MB/个） |
| 性能 | Android 中端机可实时；iOS 需注意内存 |
| 适合场景 | 仅做离线命令词/简单药品名，不适合长句听写 |

---

> **本报告结论**：爸妈宝项目语音部分 **首选科大讯飞（在线 ASR + TTS + 离线命令词）**，既有行业最好的方言兼容度，又有成熟的离线方案，是老人友好型 APP 的最稳妥选择。如果团队是 Flutter 重度用户且希望快速 MVP 验证，可选阿里云作为起手方案，后续再补充离线能力。
