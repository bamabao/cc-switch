# 爸妈宝 — 远程仓库推送指南
# ============================================================
# 说明：项目已有本地 git，按以下步骤推送到远程仓库
# ============================================================

## 方案一：GitHub（推荐）

### 1. 创建远程仓库
- 访问 https://github.com/new
- 仓库名: bamabao
- 描述: 爸妈宝 - 老人端 APP + 子女端微信小程序，药品全生命周期管理
- 公开/私有均可（推荐私有）
- 不要勾选 "Initialize with README"（本地已有）
- 点击 Create repository

### 2. 推送
```bash
# 首次推送
git remote add origin git@github.com:<你的用户名>/bamabao.git
git branch -M main
git add -A
git commit -m "🎉 初始化：爸妈宝后端API + Flutter APP 骨架

后端: FastAPI (Python 3.11) 约6200行，含用户认证/药品管理/积分商城/审核日志
APP:  Flutter 骨架已完成
CI:   GitHub Actions 配置完成
文档: 完整README/ROADMAP/API文档"
git push -u origin main
```

## 注意事项
- 确保本地 SSH Key 已添加到 GitHub/GitLab
  ```bash
  # 查看已有 key
  cat ~/.ssh/id_ed25519.pub
  # 如果没有则生成
  ssh-keygen -t ed25519 -C "your_email@example.com"
  # 然后添加到 GitHub: Settings -> SSH and GPG keys
  ```
- 首次推送前可以将 `bamabao.db`（SQLite 开发档）保留在本地，.gitignore 已屏蔽
- 后续生产用 MySQL + MinIO，各自独立维护
