# 「药快吃完了→一键补药」转化流程设计

> 交付物 4/4 · 小电 ⚡ · 2026.6.29
> 适用：爸妈宝 v1.5 商城模块（v1 提前设计流程留用）
> 注意：此模块为 v1.5 功能，当前仅做设计，不纳入 MVP v1 开发

---

## 一、设计目标

> 将"用药提醒"的自然使用场景，**无缝延伸为购药转化**。让老人在"药快吃完了"的时间节点，以最低操作成本完成补药。

---

## 二、触发时机

### 2.1 库存倒计时触发

| 时机 | 条件 | 动作 |
|------|------|------|
| 黄灯预警 | 库存 ≤ 7天 | APP内药品卡片显示「预计还能吃 %d 天」标签 |
| 红灯告警 | 库存 ≤ 3天 | 弹窗「%s 快吃完了，需要补药吗？」 |
| 空库存 | 库存 = 0天 | 首页药品变灰，点击直接进入补药流程 |

### 2.2 主动入口

| 位置 | 说明 |
|------|------|
| 药品详情页 →「补药」按钮 | 任何时刻用户可主动发起 |
| 个人中心 →「药品商城」 | 浏览模式，不针对特定药品 |
| 首页药品卡片 → 长按 | 快捷补药入口 |

### 3.2 关键设计点

**① 一键复购 vs 替换推荐**
- 默认选中「相同品牌上次购买」选项
- 下方展示 2-3 个等效低价替换药，标注省钱比例
- 不强制推送替换药，尊重用户用药习惯

**② 智能凑单**
- 当金额不满包邮标准时，展示凑单推荐
- 推荐逻辑：常用药械（创可贴、口罩、棉签）+ 历史购买记录
- 不推荐非必需保健品，避免诱导消费

**③ 库存联动**
- 下单成功后自动更新该药品库存天数
- 库存计算：购买数量 × 每盒片数 ÷ 每日用量
- 无需用户手动修改

### 4.2 家属端流程

```

  点击「帮TA补药」→ 确认药品和数量 →

    → 推送通知老人："小明帮您补了2盒络活喜，预计后天到"
```

## 五、药品商城（独立浏览 — v1.5）

### 5.1 商城分类

| 一级分类 | 二级分类 |
|---------|---------|
| 慢病用药 | 降压药、降糖药、降脂药、心脑血管 |
| 常用药械 | 血糖仪、血压计、试纸/针头 |
| 家庭常备 | 创可贴、退烧药、感冒灵、胃肠药 |
| 康复护理 | 轮椅、拐杖、护腰带、理疗仪 |
| 养生保健 | 钙片、鱼油、蛋白粉 |

### 5.2 商城设计原则

- **首屏展示"我的常用"：** 基于用户服药记录自动生成常用药品列表
- **搜索大字版：** 语音搜索 + 大字输入框
- **药品详情突出安全信息：** 禁忌、相互作用大字体展示
- **不推"神药"：** 过滤保健品类中夸大宣传的商品
- **首单优惠 + 复购折扣：** 拉新转化机制

## 六、转化漏斗监控指标

| 指标 | 说明 | 目标基准 |
|------|------|---------|
| 补药提醒到达率 | 库存告警用户中看到提醒的比例 | ≥ 90% |
| 提醒→详情页转化率 | 看到提醒后点击进入补药页 | ≥ 35% |
| 详情页→下单转化率 | 进入补药页后提交订单 | ≥ 20% |
| 下单→支付转化率 | 提交订单后完成支付 | ≥ 85% |
| 整体转化率 | 提醒→支付 | ≥ 5% |
| 替换药接受率 | 选择了等效替换药的比例 | — |
| 凑单接受率 | 使用了智能凑单的比例 | — |
