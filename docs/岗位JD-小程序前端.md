# 📱 岗位 JD：小程序前端开发工程师（微信小程序方向）

## 项目简介

**爸妈宝** — 专注老人用药管理的智慧化平台。本岗位负责**子女端微信小程序**前端开发，服务于子女用户，实现对 APP 端的远程协作管理。

---

## 一、岗位职责

1. **小程序主体架构与功能开发**
   - 搭建微信小程序整体架构，独立完成全端开发
   - 实现**药品审核中心**：子女接收老人录入的药品信息并审核/修改/补充
   - 实现**远程药品管理**：查看老人当前用药列表、调整药品时间/剂量、标记停药
   - 实现**用药数据看板**：展示用药依从率、漏服统计、按时服药趋势图表
   - 实现**积分商城管理端**：为老人兑换商品、查看兑换记录、管理收货地址
   - 实现**消息推送模块**：老人漏服/误服/异常时推送通知到子女微信

2. **后端联调与集成**
   - 对接已上线的 Python FastAPI 后端（20+ API 端点）
   - 对接微信订阅消息 / 服务通知模板

3. **体验与兼容性**
   - 遵循微信小程序设计规范，保证多机型（iPhone / Android）体验一致性
   - 小程序包体积控制（分包加载，主包 ≤ 2MB）

---

## 二、任职要求

### 硬性要求
- 2 年以上**微信小程序**开发经验，有**上线交付**的小程序项目
- 精通 WXML / WXSS / JavaScript / TypeScript，熟悉小程序生命周期与 API
- 熟悉至少一种主流框架（Taro / uni-app / 原生开发），**原生优先**
- 熟悉 RESTful API 对接、JSON/Protobuf 数据解析
- 有微信支付 / 订阅消息 / 云开发等微信生态能力接入经验

### 软性要求
- 良好的**数据可视化**意识：能基于 ECharts / AntV F2 等方案实现看板图表
- 对**信息层级设计**有理解：在有限屏幕空间内清晰呈现老人健康数据
- 有从 0 到 1 交付完整小程序的经验（非仅维护）
- 细心严谨——用药信息容错率极低，数据准确性优先
