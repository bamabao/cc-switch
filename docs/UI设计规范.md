# 爸妈宝 — 适老化 UI/UX 设计规范

> 这份文档是 APP 前端和 UI 设计师的执行标准，所有页面必须遵循。

---

## 一、全局视觉规范

### 1.1 字号

| 元素 | 字号 | 说明 |
|---|---|---|
| 页面主标题 | 56px (2.5rem→5rem) | 超大，老人第一眼看到 |
| 正文/内容 | 40px (2rem) | 比正常 APP 大 2 倍 |
| 按钮文字 | 44px (2.2rem) | 按钮内文字 |
| 输入框文字 | 40px | 输入和显示 |
| 提示/辅助 | 32px (1.6rem) | 最小，仅用于次要信息 |
| 弹窗数字 | 72px+ | 积分、时间等关键数字 |

- 行高统一 1.6 倍字号
- 字重用 Medium(500) 或 Bold(700)，不用 Light/Thin

### 1.2 配色

| 用途 | 配色 | 色值 |
|---|---|---|
| 主背景 | 暖白 | #FFFDF5 |
| 卡片背景 | 白 | #FFFFFF |
| 主文字 | 纯黑 | #1A1A1A |
| 次要文字 | 深灰 | #555555 |
| 主按钮 | 橙红 | #E85D2F |
| 主按钮文字 | 白 | #FFFFFF |
| 次按钮 | 浅黄 | #FFF3D6 |
| 次按钮文字 | 深棕 | #5C3D1A |
| 成功/确认 | 翠绿 | #2BA84A |
| 警告/提醒 | 鲜红 | #E53935 |
| 积分高亮 | 金黄 | #FFB800 |
| 分割线 | 浅灰 | #E0E0E0 |

**禁止使用：** 浅灰小字、低对比渐变、半透明文字、纯白底纯灰字

### 1.3 按钮规范

| 类型 | 最小尺寸 | 圆角 | 说明 |
|---|---|---|---|
| 主操作按钮 | 80px 高 × 全宽 | 16px | 底部固定，如"确认用药" |
| 次操作按钮 | 80px 高 × 半宽 | 16px | 如"取消" |
| 列表项按钮 | 72px 高 × 全宽 | 12px | 药品列表项 |
| 图标按钮 | 80×80px | 50% | 圆形，如语音录入 |
| 弹窗按钮 | 72px 高 | 12px | 弹窗内操作 |

- 按钮间距 ≥ 16px
- 点击反馈：按下时缩放 0.96 + 颜色加深 10%
- 禁用态：透明度 0.4 + 不可点击

### 1.4 间距与布局

- 页面左右边距：24px
- 卡片间距：16px
- 卡片内边距：24px
- 列表项高度：≥ 96px
- 页面底部留白：120px（给固定按钮栏）

## 二、交互规范

### 2.1 语音兜底机制

```
用户操作 → 三次语音识别失败 → 自动弹出超大图标应急界面
```

应急界面包含：
- 3 个超大图标按钮：📞打电话 / 🏠回首页 / ❓帮助
- 全屏高对比背景
- 语音播报："操作不成功，您可以点击打电话给家人，或回到首页"

### 2.2 弹窗规范

- 全屏弹窗（非小窗）
- 背景遮罩 90% 不透明
- 弹窗内容居中，字号 ≥ 40px
- 最多 2 个按钮（确认/取消）
- 弹窗时长 ≥ 3 秒后自动消失（用药提醒除外，需手动关闭）

### 2.3 用药提醒弹窗（特殊）

```
┌─────────────────────────┐
│                         │
│    ⏰ 该吃药了！        │
│                         │
│  药品：阿莫西林胶囊      │
│  用量：2粒              │
│  分类：内服药            │
│  注意：饭后服用，忌酒    │
│                         │
│  ⚠️ 漏服不可双倍补吃     │
│                         │
│  ┌─────────────────┐    │
│  │  ✅ 我已经吃了   │    │
│  └─────────────────┘    │
│                         │
└─────────────────────────┘
```

- 语音播报药名 + 用量（方言/普通话可切换）
- 15 分钟未确认 → 二次弹窗 + 二次语音
- 30 分钟未确认 → 推送子女

### 2.4 手势操作

- **禁止：** 滑动手势、长按、双击、缩放手势
- **仅用：** 单击（点击）
- 点击区域 ≥ 80×80px
- 防误触：同一按钮 500ms 内只响应一次

### 2.5 页面转场

- 仅用：左滑进入、右滑返回（简单直觉）
- 转场动画 ≤ 300ms
- 禁止：翻转、缩放、3D 效果

## 三、页面设计要求

### 3.1 首页

```
┌─────────────────────────────┐
│  👤 张奶奶你好！             │
│  今日积分：120  连续打卡：5天 │
├─────────────────────────────┤
│                             │
│  ⏰ 下一次用药              │
│  13:00 阿莫西林胶囊 2粒     │
│  ┌───────────────────────┐  │
│  │  📋 查看今日用药计划   │  │
│  └───────────────────────┘  │
│                             │
├─────────────────────────────┤
│  💊 我的药箱                │
│  内服 5种 | 外用 2种        │
│  ┌───────────────────────┐  │
│  │  ➕ 添加新药           │  │
│  └───────────────────────┘  │
│                             │
├─────────────────────────────┤
│  🎁 积分兑礼                │
│  当前积分：120              │
│  ┌───────────────────────┐  │
│  │  🛒 去逛商城           │  │
│  └───────────────────────┘  │
│                             │
├─────────────────────────────┤
│  📊 用药记录  |  ⚙️ 设置    │
└─────────────────────────────┘
```

### 3.2 药品录入页

- 分类选择：4 个超大图标（内服/外用/针剂/滋补）
- 录入方式：语音口述 🎤 / 手动填写 ⌨️（二选一）
- 语音录入流程：按住说话 → 识别 → 回显确认 → 提交
- 手动填写：大字段，每个字段独立一屏滚动
- 药盒拍照：超大相机按钮，拍完自动放大显示

### 3.3 积分商城页

- 商品大图（200px 高）+ 超大文字标价
- 每行 1 个商品（不要 2 列网格）
- 兑换按钮：全宽 80px 高
- 兑换成功 → 全屏庆祝弹窗 + 音效 + 语音鼓励

### 3.4 我的积分页

- 顶部：超大数字显示总积分（72px+）
- 中部：今日获得、连续打卡天数
- 底部：历史兑换记录列表

## 四、语音播报规范

| 场景 | 播报内容 | 语速 |
|---|---|---|
| 用药提醒 | "张奶奶，该吃药了。阿莫西林胶囊，2粒，饭后服用。注意，漏服不可双倍补吃。" | 慢速 0.8x |
| 积分奖励 | "今天按时吃药，获得10积分，坚持就能兑换礼品啦！" | 中速 1.0x |
| 安全警示 | "请注意，这是外用药，切勿口服。" | 中速 1.0x，语气加重 |
| 漏服告警 | "张奶奶30分钟前未确认用药，已通知家属。" | 中速 1.0x |
| 操作失败 | "操作没有成功，您可以再试一次，或点击打电话给家人。" | 慢速 0.8x |

- 语音引擎：优先选方言（粤语/四川话/上海话等），无方言则普通话
- 音量：最大音量播放（即使用户调低，提醒也要大声）
- 重复：关键提醒播报 2 遍

## 五、无障碍检查清单

- [ ] 所有文字 ≥ 32px
- [ ] 所有按钮 ≥ 80px 高
- [ ] 所有可点击区域 ≥ 80×80px
- [ ] 文字对比度 ≥ 7:1（WCAG AAA）
- [ ] 无纯灰小字
- [ ] 无复杂手势操作
- [ ] 语音识别失败有兜底
- [ ] 弹窗全屏展示
- [ ] 关键操作有二次确认
- [ ] 药品名称/剂量/时间三要素始终可见
