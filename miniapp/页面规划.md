# 爸妈宝 — 子女端微信小程序 页面规划文档

> **版本**: v1.0  
> **范围**: 子女端微信小程序一期（药品审核中心、远程药品管理、用药数据看板、积分&商城管理、消息推送）  
> **目录结构**: 全部页面位于 `pages/` 目录下，公共组件位于 `components/`，工具函数位于 `utils/`

---

## 二、逐页面详细规划

### 2.1 首页 — 用药数据看板 (`pages/index/index`)

- **路径**: `pages/index/index`
- **导航方式**: tabBar 页面（底部第1项）

#### 功能模块

| # | 区域 | 内容 | 备注 |
|---|------|------|------|
| 1 | **顶部信息** | 老人头像 + 名称 + "健康分"（四色分级） | 绑定的第一位老人默认展示 |
| 2 | **今日概览** | "今日应服 X / 已服 Y / 漏服 Z" 大字卡片 | 三组数字，Z 标红 |
| 3 | **核心统计卡片** | 4 个 stat-card 横排：本日漏服、本周漏服、临期药品、余量预警 | 点击跳转对应列表 |
| 4 | **周用药趋势** | 横柱状图：周一到周日每日应服剂量（mg/ml） | chart-bar 组件 |
| 5 | **最近待审核** | 最近 3 条待审核药品卡片（水平滚动），点"查看更多"跳转审核页 | 红点提示数量 |
| 6 | **最近消息** | 最近 2 条通知摘要，点"查看更多"跳转消息页 | |

#### 交互逻辑

- **下拉刷新**: 重新拉取全部看板数据
- **老人切换**: 顶部头像区支持左滑/右滑切换绑定的多位老人
- **空状态**: 未绑定老人时显示引导提示 → 跳转 settings 页绑定
- **待审核红点**: 首页右上角、药品 tab 同步显示待审核数量

#### API 接口

| 接口 | 用途 |
|------|------|
| `GET /api/v1/dashboard/summary` | 获取看板概要（今日/本周数据） |
| `GET /api/v1/dashboard/trend` | 获取周用药趋势数据 |
| `GET /api/v1/drug/pending?limit=3` | 最近待审核药品（首页展示用） |
| `GET /api/v1/messages/latest?limit=2` | 最近消息摘要 |

### 2.2 药品审核中心 (`pages/drug-audit/drug-audit`)

- **路径**: `pages/drug-audit/drug-audit`
- **导航方式**: 可设置 badge，从药品管理页或首页进入

#### 功能模块

| # | 区域 | 内容 | 备注 |
|---|------|------|------|
| 1 | **顶部 Tab** | 全部 / 内服 / 外用 / 针剂 — 四种分类 | 横向滑动 Tab |
| 2 | **待审核清单** | drug-card 纵向列表 | 每条显示：药品名、剂量、频次、途径、状态（待审核） |
| 3 | **全选操作栏** | 底部浮层：全选 → 批量通过 / 批量拒绝 | 仅选中待审核状态时出现 |
| 4 | **每个卡片** | 左滑露出"通过"（绿）/ "拒绝"（红）操作按钮 | 快速单条处理 |

#### 交互逻辑

- **单条审核**: 点击 drug-card 右侧"审核" → 底部弹出 ActionSheet："通过/拒绝/查看详情"
- **批量审核**: 底部工具栏勾选多条 → 点击"一键通过" → 确认弹窗 → 接口提交 → 成功 Toast
- **空状态**: "暂无待审核药品 🎉" 带刷新按钮
- **审核后动画**: 通过时卡片飞入"已通过区"动画，拒绝时灰化并保留在原处带叉标记
- **已审回看**: 顶部 Tab 栏加"已审核"分类，可查看历史操作记录

#### API 接口

| 接口 | 用途 |
|------|------|
| `GET /api/v1/drug/audit-list?status=pending&type=all` | 获取待审核列表（支持分类筛选） |
| `POST /api/v1/drug/audit/batch` | 批量审核（参数：药品ID数组 + 操作：approve/reject） |
| `POST /api/v1/drug/audit/single` | 单条审核 |
| `GET /api/v1/drug/audit-list?status=done` | 已审核历史 |

- **路径**: `pages/drug-manage/drug-manage`
- **导航方式**: tabBar 页面（底部第2项）

#### 功能模块

| # | 区域 | 内容 | 备注 |
|---|------|------|------|
| 1 | **顶部搜索栏** | 搜索框：按药品名模糊搜索 + 分类筛选（全部/内服/外用/针剂） | |
| 2 | **状态筛选** | 全部 / 审核中 / 已生效 / 已停用 — 横向滚动的 pill 按钮 | |
| 3 | **药品列表** | drug-card 纵向列表，右上角状态标签（审核中/已生效/已停用） | 已生效药品显示剩余量/有效期 |
| 4 | **FAB 按钮** | 右下角悬浮 "+" 按钮 → 新增药品 | 点击跳转 drug-form 页 |
| 5 | **长按操作** | 长按卡片弹出操作菜单：编辑 / 停用 / 删除 | 删除二次确认 |

#### 交互逻辑

- **点击药品卡片**: 跳转 `drug-detail` 药品详情页
- **搜索联动**: 输入即搜索（防抖 500ms），搜索时列表过滤
- **筛选联动**: 切换分类/状态 Tab 重新请求列表
- **下拉刷新&上拉加载**: 分页加载，每页 20 条

#### API 接口

| 接口 | 用途 |
|------|------|
| `GET /api/v1/drug/list?page=1&pageSize=20` | 药品列表（支持search/type/status参数） |
| `PUT /api/v1/drug/deactivate` | 停用药品 |
| `DELETE /api/v1/drug/delete` | 删除药品 |

### 2.4 新增/编辑药品表单 (`pages/drug-form/drug-form`)

- **路径**: `pages/drug-form/drug-form`
- **导航方式**: 从药品管理页 FAB 或编辑按钮进入

#### 功能模块

| # | 表单字段 | 类型 | 说明 |
|---|---------|------|------|
| 1 | **药品名称** | 输入框 | 必填 |
| 2 | **药品分类** | 单选：内服/外用/针剂 | 必选，选不同分类影响下方剂量单位 |
| 3 | **规格** | 输入框 | 如 "0.25g × 24片" |
| 4 | **给药途径** | 单选：口服/舌下/注射/雾化/涂抹... | 动态根据分类决定默认选项 |
| 5 | **单次剂量** | 数字输入 + 单位选择 | 片/ml/粒/支/贴/spray — 根据分类联动 |
| 6 | **用药频率** | 组合：每天 X 次 / 每次 Y 个 / 固定时间 | 支持自定义频次文本 |
| 7 | **用药时间** | 时间选择器（可多选，可选时段如"餐前/餐后"） | 如 "08:00、12:00、18:00" |
| 8 | **药品总数** | 数字输入 | 库存总数，用于余量预警 |
| 9 | **到期日期** | 日期选择器 | 用于临期预警 |
| 10 | **备注** | 多行文本输入 | 选填（如医生嘱托、过敏提示） |
| 11 | **药品图片（可选）** | 图片上传（wx.chooseMedia） | 最多3张，拍摄药盒照片 |

#### 交互逻辑

- **编辑模式**: 页面 OnLoad 参数 `?id=xxx` 表示编辑，自动填充表单
- **表单校验**: 提交前逐字段校验，错误字段标红提示
- **保存即提交审核**: 子女端新增/编辑药品 → 保存即进入"待审核"状态 → 自动推送给老人端复核
- **草稿暂存**: 支持本地 Storage 暂存未提交表单
- **提交成功**: 跳转回 drug-manage 列表页并显示 Toast "已提交审核"

#### API 接口

| 接口 | 用途 |
|------|------|
| `POST /api/v1/drug/create` | 新增药品（创建审核项） |
| `PUT /api/v1/drug/update` | 编辑药品（更新后需老人重新审核） |

## 四、数据流 & 路由关系

```
首页 (index)
 ├─ 消息卡片 → message-list（筛选消息类型）
 ├─ 待审核卡片 → drug-audit（"查看更多"）
 ├─ 药品余量/临期 → drug-detail（跳转具体药品）
 └─ 设置齿轮 → settings

药品管理 (drug-manage)
 ├─ 点击卡片 → drug-detail
 ├─ 编辑 → drug-form?id=xxx
 ├─ 新增FAB → drug-form
 └─ badge → drug-audit（待审核入口）

药品审核 (drug-audit)
 └─ 点击详情 → drug-detail

积分商城 (points-mall)
 ├─ 积分明细 → points-history?tab=points
 ├─ 兑换记录 → points-history?tab=exchange
 └─ 我的订单 → order-list

订单 (order-list)
 └─ 点击订单 → order-detail

消息列表 (message-list)
 └─ 点击消息 → message-detail（或按类型跳转对应页面）
      ├─ 漏服/过期/余量 → drug-detail
      ├─ 审核通知 → drug-audit
      ├─ 打卡/积分 → points-mall
      └─ 订单 → order-detail
```

## 五、UI/UX 规范建议

| 规范项 | 建议 |
|--------|------|
| **主色调** | `#4A90D9`（信任蓝） — 医药健康主色 |
| **辅助色** | 红 `#E74C3C`（漏服/预警）、绿 `#2ECC71`（已服/通过）、橙 `#F39C12`（临期/提醒） |
| **字号规范** | 标题 18px/17px，正文 15px，副文 13px，标签 11px |
| **圆角** | 卡片 12px，按钮 8px，头像 50% |
| **阴影** | 卡片轻阴影 `0 2px 8px rgba(0,0,0,0.08)` |
| **加载状态** | 骨架屏 Skeleton 组件（首页和列表页必须），其余页面用 Loading |
| **错误处理** | 全局统一错误 Toast + 网络异常重试按钮 |
| **无障碍** | 所有按钮标注 aria-label，核心操作字号支持动态缩放 |
| **离线体验** | 首页看板数据做 Storage 缓存，无网络时展示上次数据 + 提示"离线数据" |
