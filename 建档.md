# 爸妈宝 APP — 项目档案

## 基本信息

- **项目名称：** 爸妈宝
- **类型：** 老人端 APP + 子女端微信小程序 + Python 后端
- **开发周期：** 4 个月
- **定位：** 一期仅实现药品全生命周期管理（内服、外用、针剂、滋补辅药）
- **核心设计理念：** 超大字体极简界面 + 用药积分激励商城 + 子女远程审核机制

## 一、目标用户

| 端 | 用户 | 核心场景 |
|---|---|---|
| 老人端 APP | 60+ 老人（老花眼/视力衰退） | 自行/语音录制用药、接收提醒、确认用药、赚积分兑礼品 |
| 子女端小程序 | 老人子女 | 审核药品、远程管理、监控用药数据、管理商城订单 |

## 二、完整功能模块分解

### （一）四大用药分类录入

#### 1. 内服药
- 剂型：药片、胶囊、冲剂、口服液、中药汤剂
- 填写字段：药品名称、生产厂家、保质期、单次剂量（片/粒/ml/袋）、每日频次、服用时段、饭前/饭后/空腹、忌口、不良反应、总剩余药量

#### 2. 外用药
- 剂型：药膏、喷雾、滴眼液、贴剂、碘伏、洗剂
- 填写字段：使用部位、单次用量、涂抹/敷贴周期、皮肤过敏提示、存放要求

#### 3. 针剂注射
- 类型：胰岛素、皮下注射针、长效药剂、输液
- 填写字段：药剂名称、注射剂量(IU/ml)、注射部位、注射周期（每日/隔日/每周）、注射前摇匀提示、低血糖/过敏风险警示

#### 4. 滋补辅药
- 降压滋补品、保健调理品 → 单独归档区分

### （二）精细化剂量配置
- 可视化刻度：半片、1/4片、1/2ml、5ml、10ml + 量杯/药片示意图
- 语音口述录入，自动识别保存
- 剂量修改 → 自动进入待审核状态 → 未审核不生效

### （三）多层用药定时提醒
- 到期超大弹窗 + 方言/普通话语音播报（药品种类、用量）
- 超时15分钟 → 二次提醒
- 超时30分钟未确认 → 推送小程序告警给子女
- 安全语音警示（漏服不可双倍补吃、针剂禁止过量、外用药切勿口服）
- 用药完成一键确认 → 自动记录存档 → 即时发放积分

### （四）药品档案管理
- 药盒/说明书拍照存档 + 放大查看
- 余量预警（剩3天 → 推送采购提醒）
- 过期预警（临近过期 → 首页置顶提示）
- 完整历史台账（内服/外用/打针分类展示，永久云端存储，不可删除）

### （五）药品提交 & 子女审核机制（核心安全规则）

**老人端：**
- 新增/修改药品 → 提交后状态 = "待审核"
- 未审核通过 → 不生成提醒、不弹窗播报

**子女端小程序：**
- 审核通过 → 同步至老人APP，提醒生效
- 驳回修改 → 填写驳回原因，退回老人重编
- 子女远程新增 → 直接录入并一键审核通过

**全程操作留痕：**
- 新增/修改/提交/通过/驳回 → 全部云端存档，子女随时溯源

### （六）积分激励兑换商城

**积分获取：**
- 单次完成用药 → 10积分
- 连续7天无漏服 → 额外50积分
- 连续30天完整打卡 → 200积分
- 完整录入档案/清理过期药 → 小额积分
- 漏服不扣分（仅取消当日积分）

**积分商城：**
- 首页底部超大「积分兑礼」按钮
- 商品：日用类（防滑袜/老花镜/水杯/按摩梳/毛巾）、健康类（血压计袖带/药盒/护膝/艾草贴）、文娱类（戏曲U盘/大字扑克牌/益智玩具）
- 兑换流程：点击→一键兑换→后台登记→子女小程序查看订单/物流
- 积分永久不清零

**积分可视化：**
- 独立「我的积分」页面（大字：总积分、今日积分、连续打卡天数、历史兑换记录）
- 每日用药后暖心弹窗 + 语音鼓励

### （七）超大简易界面规范
- 全局字号≥2倍正常，按钮尺寸≥80px
- 高对比配色（黑字黄底/白底黑字）
- 页面仅保留核心按钮，删减多余文字/弹窗/广告
- 所有弹窗/提醒/积分提示 → 全屏大字展示
- 语音识别兜底：三次失败 → 超大图标应急操作界面

### （八）子女端微信小程序一期功能
1. 药品审核中心 ← 汇总待审核药品
2. 远程药品管理 ← 新增/编辑/删除，一键审核生效
3. 用药数据看板 ← 日/周记录、漏服、过期/余量预警
4. 积分&商城管理 ← 查看积分、兑换记录、订单物流
5. 消息推送 ← 漏服/待审核/过期/连续打卡通知

## 三、技术架构（一期）

| 层 | 技术选型 | 说明 |
|---|---|---|
| 后端 | Python | RESTful API |
| 数据库 | MySQL 8.0 | 药品信息、用药记录、审核日志、积分流水、商城订单、用户绑定——分层存储 |
| 老人端 | 待定（原生/跨平台） | 需高标准适老化设计 |
| 子女端 | 微信小程序 | 配套审核/管理/监控 |
| 数据同步 | 实时双向 | 老人APP ↔ 后端 ↔ 小程序 |
| 安全 | 加密存储 | 所有用药/积分数据 |

## 四、设计原则
1. **安全第一** — 子女审核机制为用药安全核心防线
2. **适老优先** — 字号/配色/按钮/弹窗/语音兜底
3. **正向激励** — 只奖不罚，积分永久有效
4. **数据不可丢** — 永久云端存储，全操作留痕可溯源
5. **零硬件依赖** — 纯软件方案

## 五、边界 & 排期
- 一期无硬件、无棋牌、无家电控制
- 开发周期：4个月
